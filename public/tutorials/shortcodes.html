O que são Shortcodes

Shortcodes são códigos curtos envoltos em colchetes (ex: [meu-shortcode]) que o administrador de conteúdo digita em blocos de texto no editor de postagens ou páginas. No carregamento da página pública, o sistema intercepta esses códigos e os substitui por blocos complexos de HTML dinâmico de forma automática.

Registrando um Shortcode

Utilize a classe helper ContentHelper::registerShortcode() dentro do método boot() do Service Provider do seu plugin para cadastrar a tag. O método aceita parâmetros sequenciais adicionais para autodocumentação na administração:

PHPContentHelper::registerShortcode(
    string $tag,
    callable $callback,
    string $description = '',
    string $example = '',
    array $attributes = []
);

Exemplo de cadastro de shortcode básico simplificado (tag auto-fechada) utilizando parâmetros sequenciais:

PHPuse App\Helpers\ContentHelper;

// Cadastro da tag [video id="X"]
ContentHelper::registerShortcode(
    'video',
    function($attributes, $content) {
        $id = $attributes['id'] ?? null;

        if (!$id) {
            return '<!-- Erro: [video] requer atributo "id" -->';
        }

        return view('video::player', ['id' => $id])->render();
    },
    'Renderiza um player de vídeo do YouTube ou Vimeo.',
    '[video id="dQw4w9WgXcQ"]'
);

Esquema de Atributos (Schema)

Você pode declarar o quinto parâmetro de atributos aceitos pelo seu shortcode de forma estruturada. Esses dados alimentam a página de Referência de Shortcodes na Admin e constroem de forma automática o formulário do TinyMCE na hora de inserir ou editar o recurso, evitando erros de digitação.

Os itens configurados no array de atributos suportam as seguintes chaves de definição:

  • label: Rótulo amigável que aparecerá ao lado do campo no formulário do editor.
  • type: O tipo do campo (text, number, select, textarea, checkbox).
  • placeholder: Texto de ajuda exibido dentro de inputs de texto vazios.
  • options: Array de chave e valor (usado exclusivamente quando o tipo for select).
  • default: O valor padrão inicial com o qual o formulário será pré-preenchido.
  • required: Sinaliza se o preenchimento do campo é obrigatório para gerar o shortcode.

Sintaxe de Uso no Painel

As tags de shortcode cadastradas podem ser escritas pelos editores de conteúdo de três formas distintas no editor:

Estrutura Sintaxe de Exemplo Utilidade
Simples [meu-componente] Inserção de widgets estáticos globais (ex: formulário de newsletter).
Com Atributos [meu-componente id="42" estilo="dark"] Carrega um modelo específico do banco parametrizado (ex: mapa de ID 42).
Com Conteúdo [meu-componente]Conteúdo Interno[/meu-componente] Aplica formatação ou envoltórios especiais ao redor do texto digitado.

Exemplo Prático (FAQ)

Este exemplo prático mostra como o plugin de FAQ registra seu shortcode passando os parâmetros de forma sequencial para carregar conjuntos de perguntas e respostas serializados nas Options do banco, configurando o esquema de atributos para exigir o preenchimento obrigatório da chave (slug) na interface do TinyMCE:

PHPuse App\Helpers\ContentHelper;

ContentHelper::registerShortcode(
    'faq',
    function($attributes, $content) {
        $slug = $attributes['slug'] ?? null;

        if (!$slug) return '';

        // Busca os dados serializados da tabela options
        $optionData = getOption("faq_{$slug}");
        if (!$optionData) return '';

        // Deserializa e renderiza a view correspondente
        $faq = is_string($optionData) ? json_decode($optionData, true) : $optionData;

        if (empty($faq) || empty($faq['items'])) return '';

        return view('faq::public.show', compact('faq'))->render();
    },
    'Carrega e exibe um conjunto de Perguntas e Respostas estruturado.',
    '[faq slug="vendas"]',
    [
        'slug' => [
            'label'       => 'Slug identificador da FAQ',
            'type'        => 'text',
            'placeholder' => 'Ex: vendas, suporte, geral',
            'required'    => true
        ]
    ]
);

Remoção de Tags de Parágrafo

Comportamento Automatizado: Editores ricos (como o TinyMCE) frequentemente envolvem os shortcodes escritos em tags de parágrafo durante a digitação (ex: <p>[meu-shortcode]</p> ou <p>[meu-shortcode]conteúdo[/meu-shortcode]</p>). O renderizador de conteúdo do Lunar Base limpa e remove essas tags automaticamente em segundo plano para que as visões HTML injetadas não sofram com margens ou quebras de layout indesejadas geradas pelas tags parágrafo ao redor.
