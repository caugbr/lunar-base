Introdução

O sistema de temas do Lunar Base permite transformar completamente a identidade visual do site público. Os temas herdam a lógica do core por padrão, mas dão total liberdade para que qualquer arquivo visual seja substituído de forma transparente.

Estrutura do Tema

Os temas residem no diretório /themes na raiz do projeto. Um tema chamado "AstralDark" possui a seguinte estrutura:

themes/
└── AstralDark/
    ├── resources/
    │   ├── views/
    │   │   └── public/
    │   │       ├── page-templates/
    │   │       │   └── default.blade.php  {{-- Template de Páginas --}}
    │   │       └── post-templates/
    │   │           └── default.blade.php  {{-- Template de Posts --}}
    │   └── assets/
    │       ├── css/
    │       │   └── style.css
    │       ├── js/
    │       │   └── app.js
    │       └── images/
    │           └── screenshot.png
    ├── theme.json
    └── ThemeServiceProvider.php            {{-- Opcional (Lógica avançada) --}}

O Manifesto (theme.json)

O arquivo theme.json define as informações que aparecem na biblioteca de temas do painel de administração:

JSON{
    "name": "Astral Dark",
    "description": "Um tema focado em tons escuros e constelações.",
    "version": "1.0.0",
    "author": "Equipe Lunar",
    "screenshot": "resources/assets/images/screenshot.png"
}

Sobrescrita de Views (Overrides)

O motor do Lunar Base prioriza as views do seu tema ativo de forma transparente. Se você colocar um arquivo dentro do diretório do tema que corresponda ao caminho das views do core, o sistema carregará o seu arquivo no lugar do original.

Hierarquia de Carregamento:

Ao renderizar um Post, o Laravel busca o arquivo na seguinte ordem:

  • 1º: /themes/NomeDoTema/resources/views/public/post-templates/default.blade.php (Tema ativo)
  • 2º: /resources/views/public/post-templates/default.blade.php (Fallback do core)

Substituindo o Layout Geral do Site:

Para mudar a estrutura inteira do site (como o Header e o Footer), crie o arquivo em:

/themes/NomeDoTema/resources/views/public/site-layout.blade.php

Assets e Links Simbólicos

Os arquivos estáticos (CSS, JS, imagens) do seu tema ficam isolados e são disponibilizados publicamente através de links simbólicos gerados na pasta pública do servidor.

Referenciando Assets no Blade:

BLADE@once
@push('styles')
<link rel="stylesheet" href="{{ asset('themes/astraldark/css/style.css') }}">
@endpush
@once
Gerando links manualmente: Ao criar ou copiar um tema manualmente em disco, você pode forçar a regeneração do link simbólico executando o comando no console: 
BASHphp artisan themes:link NomeDoTema

Boas Práticas de Design

  • Uso de @once: Envolva as chamadas de folhas de estilo e scripts JS nas tags @once e @push para prevenir carregamentos duplicados em loops ou acordeões.
  • Não edite o Core: Se precisar alterar o design de uma view padrão do sistema, copie-a para a pasta correspondente no seu tema e faça a edição por lá.
  • Imagens Otimizadas: Prefira formatos como WebP ou SVG para manter o carregamento das páginas rápido e responsivo.
