Introdução

O Lunar Base possui dois caminhos de persistência leve de informações sem a necessidade de criar tabelas dedicadas: a API de Settings (voltada para formulários administrativos) e a API de Options (voltada para chaves de dados internas).

Settings (Painel Admin)

A classe Settings permite injetar de forma declarativa novos campos na tela de configurações gerais do site no painel administrativo.

1. Criando um Novo Grupo (Aba):

PHPuse App\Support\Settings;

Settings::addGroup('meu_grupo', [
    'title'       => 'Meu Módulo',
    'description' => 'Configurações de integração e comportamento',
    'icon'        => 'cpu', // Ícone Lucide
]);

2. Injetando Campos no Grupo:

PHPSettings::add([
    'key'         => 'meu_modulo_ativo',
    'type'        => 'switch',
    'active'      => 'Ativado',
    'inactive'    => 'Desativado',
    'label'       => 'Habilitar Módulo',
    'description' => 'Ativa ou desativa as integrações do módulo.',
    'default'     => true,
], 'meu_grupo');

3. Recuperando os Valores no Front-end:

BLADE@if(setting('meu_grupo.meu_modulo_ativo', true))
    <div>O módulo está ativo!</div>
@endif

Campos Suportados em Settings

A API de Settings aceita uma variedade de tipos de inputs que o núcleo renderiza automaticamente com validação nativa:

Tipo (type) Parâmetros Extras Descrição
'text' — Input de texto convencional de linha única.
'textarea' — Área de texto de múltiplas linhas para descrições.
'number' 'attributes' => ['min' => 1, 'max' => 10] Input numérico controlado por limites.
'switch' 'active', 'inactive' Interruptor liga/desliga deslizante estético.
'icon' 'can_clear' => true Seletor de ícones Lucide interativo com busca.
'select' 'options' => ['k' => 'v'] Dropdown para seleção de opções cadastradas.
'subtitle' — Separador estético visual para agrupar campos.

Options (Uso Interno)

A API de Options é ideal para gravar dados estruturados (como arrays, senhas criptografadas ou estruturas JSON) de forma direta no banco de dados, sem precisar expô-los em páginas de configurações.

Métodos Globais de Controle:

PHP// 1. Criando ou atualizando um registro (converte arrays em JSON automaticamente)
setOption('faq_geral_dados', ['titulo' => 'FAQ Geral', 'itens' => []]);

// 2. Recuperando o valor convertido de volta de forma tipada
$faq = getOption('faq_geral_dados'); // Retorna o array PHP

// 3. Salvando dados criptografados (senhas de API, chaves privadas)
setOption('api_token', 'chave-super-secreta', 'password'); // Encripta no banco via Crypt()
$token = getOption('api_token'); // Retorna a chave descriptografada automaticamente

// 4. Buscando coleções por prefixo (Reduz queries)
$faqs = getPrefixedOptions('faq_'); // Retorna todas as chaves que começam com "faq_"

Tabela Comparativa

Característica Settings Options
Formulário Automático ✅ Sim (Gerado declarativamente) ❌ Não
Validação de Input ✅ Sim (Nativa do controlador) ❌ Não (Validar manualmente se necessário)
Criptografia em chaves ❌ Não ✅ Sim (Usando o tipo 'password')
Armazena Objetos/Arrays ❌ Apenas chaves escalares simples ✅ Sim (Casting automático para JSON)
