Introdução

O sistema de temas do Lunar Base CMS é projetado para ser leve, modular e flexível. Por padrão, ele herda a estrutura visual básica do sistema, mas permite que o desenvolvedor reescreva layouts parciais ou inteiros e execute lógicas robustas de negócios, transformando o visual do site público de forma limpa.

View Override

Substitua qualquer página, post ou template global do núcleo apenas inserindo o arquivo correspondente.

Lógica PHP

Opcionalmente, use Service Providers para estender o painel, registrar rotas ou escutar hooks.

Settings Integradas

Injete parâmetros cosméticos (cores, layouts) no painel do Admin e dê controle ao administrador.

Assets Isolados

Estrutura de arquivos CSS, JS e imagens isolada em cada tema com carregamento otimizado.

Estrutura de um Tema

A pasta de temas está localizada na raiz do projeto em /themes. Um tema típico segue o seguinte mapa de arquivos:

themes/
└── NomeDoTema/
    ├── resources/
    │   ├── views/
    │   │   └── public/
    │   │       ├── templates/
    │   │       │   └── default.blade.php  {{-- Template de Páginas --}}
    │   │       └── post-templates/
    │   │           └── default.blade.php  {{-- Template de Posts --}}
    │   └── assets/
    │       ├── css/
    │       │   └── style.css
    │       ├── js/
    │       │   └── app.js
    │       └── images/
    │           └── screenshot.png
    ├── theme.json
    └── ThemeServiceProvider.php            {{-- Opcional (Lógica avançada) --}}

Arquivos Principais:

Arquivo Descrição
theme.json Manifesto com metadados do tema (autor, versão, imagem de preview)
ThemeServiceProvider.php Classe PHP opcional para injetar comportamentos avançados
resources/views/ Templates Blade que substituem ou estendem o visual padrão do core

Criando seu Primeiro Tema

1Use o Comando Artisan

BASHphp artisan theme:create "NomeDoTema" "Descrição do tema"

O console irá:

  • Criar a estrutura completa de pastas do tema em /themes/NomeDoTema
  • Gerar o arquivo theme.json pré-configurado
  • Criar um arquivo de view de exemplo para páginas
  • Criar o link simbólico na pasta pública public/themes/nomedotema

2Ative o Tema

Acesse o menu Admin → Aparência (Temas). O painel listará o seu tema com a imagem de preview. Basta ativá-lo para que suas views e lógicas entrem em ação.

Manifesto (theme.json)

O arquivo theme.json define as informações administrativas que serão mostradas na biblioteca de temas do Lunar Base Admin.

JSON{
    "name": "Astral Dark",
    "description": "Um tema focado em tons escuros e constelações.",
    "version": "1.0.0",
    "author": "Equipe Lunar",
    "screenshot": "resources/assets/images/screenshot.png"
}
Screenshot: Garanta que a imagem de preview declarada em screenshot exista fisicamente na pasta do tema para carregar no painel do administrador.

Sobrescrita de Views (View Override)

O Lunar Base CMS prioriza de forma transparente as views do tema ativo. Sempre que você colocar um arquivo dentro do diretório do tema que corresponda ao caminho das views do core, o sistema o usará no lugar do original.

Como funciona a hierarquia de carregamento:

Quando o usuário acessa um Post do blog, o Laravel procurará a view na seguinte ordem:

  • 1º: /themes/NomeDoTema/resources/views/public/post-templates/default.blade.php
  • 2º: /resources/views/public/post-templates/default.blade.php (Core fallback)

Substituindo o Layout Geral do Site:

Para mudar a estrutura inteira do site (como o Header, Footer e CSS globais), basta criar um arquivo em:

/themes/NomeDoTema/resources/views/public/site-layout.blade.php

Esse arquivo irá interceptar e substituir o layout original do sistema sem modificar o núcleo.

Lógica Avançada (ThemeServiceProvider)

Os temas não precisam se limitar a HTML e CSS estáticos. Ao criar a classe opcional ThemeServiceProvider.php na raiz da pasta do seu tema, você ganha acesso total aos recursos do framework no boot do sistema.

Assinatura Básica do Provedor:

PHPnamespace Themes\NomeDoTema;

use Illuminate\Support\ServiceProvider;

class ThemeServiceProvider extends ServiceProvider
{
    public function register(): void
    {
        // Registro de bindings em tempo de execução
    }

    public function boot(): void
    {
        // Lógicas executadas antes de renderizar as páginas
        // Nota: O carregamento de views do seu tema é realizado automaticamente pelo core!
    }
}
Registro Automático de Views: O core do Lunar Base cuida de prepender e indexar as views do seu tema de forma transparente. Você não precisa escrever códigos de carregamento de views no método boot(), deixando seu ServiceProvider focado unicamente na lógica de negócio.

Injeção de Configurações no Tema

Você pode expor configurações cosméticas diretamente no painel administrativo geral do site. Quando seu tema é ativado, esses campos de formulário são injetados no Admin, e se tornam inativos ou ocultos quando o tema é desligado.

Injetando campos estéticos no ServiceProvider do Tema:

PHPuse App\Support\Settings;

public function boot(): void
{
    // Adiciona um divisor estético na aba de configurações Gerais
    Settings::add([
        'type' => 'subtitle',
        'label' => 'Aparência: Astral Dark',
    ], 'general');

    // Injeta um seletor de cores para que o Administrador mude a cor do tema no painel
    Settings::add([
        'key' => 'astral_dark_accent_color',
        'type' => 'select',
        'options' => [
            'cyan' => 'Azul Celestial',
            'gold' => 'Dourado Estelar',
        ],
        'label' => 'Cor de Destaque',
        'description' => 'Escolha a cor principal para links e botões do site.',
        'default' => 'cyan',
    ], 'general', 'Aparência: Astral Dark');
}

Integração com Hooks

Os temas podem escutar e injetar HTML nos hooks definidos pelo core ou pelos plugins, e também podem definir seus próprios ganchos em seus arquivos estruturais.

Exemplo 1: Tema escutando e injetando HTML no Head do site:

PHPuse App\Support\HookManager;

public function boot(): void
{
    HookManager::register('main.head', function() {
        return '';
    }, 'AstralDark');
}

Boas Práticas

1. Organização e Isolamento:

  • Use PascalCase para pastas de temas (AstralDark, e não astral-dark).
  • Sempre referencie assets usando o caminho correto mapeado pelo link simbólico.
  • Nunca edite arquivos da pasta /resources/views do core caso queira mudar o visual. Sobrescreva-os na pasta do seu tema.

2. Compatibilidade e Fallbacks:

  • Ao ler chaves de configuração personalizadas via setting(), sempre forneça um valor padrão de fallback como segundo parâmetro para evitar quebras se o valor estiver em branco.
  • Utilize o suporte nativo do componente de Hooks com slots padrão para manter fallbacks visuais se o plugin associado for desativado.

3. Performance:

  • Sempre utilize a estrutura @once e @push para os links de CSS e JS dos seus templates. Isso evita sobrecarregar a página com arquivos repetidos caso o mesmo bloco apareça mais de uma vez.
  • Prefira imagens em formatos otimizados como WebP e SVG para compor os elementos estáticos do seu tema.
