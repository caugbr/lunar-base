O que são Rotas Dinâmicas

No desenvolvimento de plugins, registrar rotas fixas no arquivo routes.php do Laravel é ideal para o painel administrativo. Porém, no site público, muitas vezes precisamos criar caminhos amigáveis e estruturados que são gerenciados de forma dinâmica de acordo com os cadastros efetuados (como caminhos de portfólios, links de formulários ou termos do blog).

Como Registrar

Utilize a classe helper DynamicRoutes::register() dentro do método boot() do seu Service Provider de plugin para cadastrar o mapeamento:

PHPuse App\Support\DynamicRoutes;

// Registro da rota dinâmica /portfolio/{slug}
DynamicRoutes::register('GET', 'portfolio/{slug}', function($slug) {
    // Busca os dados e devolve uma resposta visual
    $item = \Plugins\Portfolio\Models\Portfolio::where('slug', $slug)->firstOrFail();

    return view('portfolio::public.show', compact('item'));
});

Como o Core Resolve as URLs

Ao receber uma requisição no site público, o motor do Laravel tenta encontrar uma rota estática convencional cadastrada no sistema. Se nenhuma rota física corresponder ao endereço acessado, o roteamento global do Lunar Base direciona a requisição para o RouteOrchestratorController:

  • 1. Captura da URL: O controlador analisa o caminho requisitado pelo visitante.
  • 2. Varredura Dinâmica: O orquestrador compara a URL com a coleção de caminhos dinâmicos registrados pelos plugins ativos em memória através de regex.
  • 3. Execução do Callback: Assim que encontra uma correspondência, ele executa o callback passando as variáveis de url associadas (ex: $slug) e renderiza a resposta de forma transparente, sem redirecionamentos perceptíveis para o navegador do visitante.

Exemplos de Registro

Exemplo 1: Rota Simples sem Parâmetros

PHPDynamicRoutes::register('GET', 'vagas-trabalho', function() {
    return view('vagas::public.lista');
});

Acessível em: https://seusite.com/vagas-trabalho

Exemplo 2: Rota Dinâmica com Múltiplos Parâmetros

PHPDynamicRoutes::register('GET', 'loja/{categoria}/{produto_slug}', function($categoria, $slug) {
    $produto = \Plugins\Loja\Models\Produto::where('slug', $slug)->firstOrFail();
    return view('loja::public.produto', compact('produto'));
});

Acessível em: https://seusite.com/loja/eletronicos/celular-smartphone

Regras de Precedência

Importante sobre Conflitos: Como as rotas dinâmicas são resolvidas de forma preguiçosa (fallback) no fim da pilha de carregamento do framework, qualquer rota estática de roteamento fixo convencional do Laravel terá prioridade. Se houver um arquivo de visão física padrão ou rota fixa com o mesmo nome exato da rota dinâmica, o Laravel resolverá a rota estática primeiro.
