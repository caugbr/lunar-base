Introdução

Os plugins oficiais do Lunar Base são módulos nativos que estendem o funcionamento do seu site. Eles já vêm instalados e, na maioria dos casos, funcionam de forma automática assim que ativados.

Este manual é voltado para quem vai utilizar os plugins: administradores, editores de conteúdo e desenvolvedores de tema.

Plug-and-Play

Ative o plugin e ele já começa a funcionar automaticamente nos locais apropriados.

Configurável

Cada plugin expõe opções no painel administrativo para personalizar comportamento.

Integrável

Hooks e shortcodes permitem incorporar os recursos em qualquer lugar do tema.

1. Avatars (Imagens de Perfil)

Permite que usuários façam upload de fotos de perfil personalizadas. A imagem é automaticamente recortada em 150×150px, convertida para WebP e exibida em todo o site.

Como funciona

  • Upload de imagens de até 5MB
  • Recorte automático em formato quadrado
  • Conversão para WebP (qualidade 90)
  • Fallback automático para Gravatar se não houver foto
  • Exclusão automática ao remover o usuário

Onde aparece

Local Comportamento
Header do admin Miniatura circular do usuário logado
Página de Perfil Card com upload e preview
Comentários Avatar do autor (se Comments ativo)

Como usar no tema

Exibindo avatar

<img src="{{ \Plugins\Avatars\Helpers\AvatarHelper::getUrl($user) }}"
     alt="Avatar"
     class="avatar-circle-sm">
Dica: o helper aceita tanto um objeto User quanto uma string de e-mail (útil para visitantes).

Hooks utilizados

  • admin.header_user_avatar — Header do painel
  • profile.after_card — Página de perfil

2. Comments (Comentários Aninhados)

Sistema completo de comentários com respostas aninhadas, moderação de spam e integração com avatares.

Como funciona

  • Aparece automaticamente ao final de cada post
  • Visitantes comentam com nome e e-mail; usuários logados com um clique
  • Respostas aninhadas ilimitadas
  • Moderação automática de palavras suspeitas

Como usar

Não precisa fazer nada! O plugin se conecta automaticamente ao hook post.footer_end.

Moderação

Comentários com estas palavras vão para moderação:

buy, viagra, free casino, spam, cryptocurrency, click here

Hook utilizado

post.footer_end

3. FAQ (Perguntas Frequentes)

Crie blocos de perguntas e respostas em formato de acordeão, usando elementos nativos HTML5.

Como criar

1Acesse Admin → Temas → FAQ

2Clique em "Novo FAQ"

Preencha Título e Slug (ex: duvidas-gerais)

3Adicione perguntas

Clique em "Adicionar Pergunta" e preencha enunciado e resposta

4Copie o shortcode

Como exibir

Shortcode

[faq slug="duvidas-gerais"]
Acessibilidade: usa <details> e <summary> do HTML5 — funciona sem JavaScript!

4. Forms (Construtor de Formulários)

Crie formulários personalizados para capturar leads e mensagens. Cada envio pode ser notificado por e-mail.

Como criar

1Admin → Formulários → Novo Formulário

2Dados básicos

  • Slug: identificador (ex: contato)
  • Título: cabeçalho do formulário
  • E-mail de destino: para notificações

3Construa os campos

Adicione campos com tipo, label, regras de validação, etc.

Tipos de campos

Tipo Uso
text Texto curto
textarea Texto longo
email E-mail validado
number Número
select Dropdown
radio Múltipla escolha (radio)
checkbox Múltipla escolha (checkbox)
switch Toggle Sim/Não

Como exibir

Shortcode

[form slug="contato"]
Atenção: para receber e-mails, configure SMTP em Admin → Configurações → E-mail.

5. Maintenance (Modo de Manutenção)

Bloqueia o acesso público ao site exibindo uma página de "em manutenção".

Como ativar

1Admin → Configurações → Manutenção

2Marque "Ativar modo de manutenção"

3Personalize a página

Ícone, título, mensagem, footer

Quem acessa durante manutenção

  • Admin e editores: navegam normalmente
  • Rotas /admin, /login: continuam funcionando
  • Visitantes: veem apenas a página de manutenção
SEO: retorna status HTTP 503 — o Google não indexa a página de manutenção.

6. Maps (Mapas Interativos)

Crie mapas OpenStreetMap com marcadores personalizados e áreas destacadas.

Como criar

1Admin → Mapas → Novo Mapa

2Configure o centro

Busque endereço ou ajuste lat/lng manualmente

3Adicione marcadores

Clique no mapa ou use "Novo marcador no centro"

Como exibir

Exibir mapa

[map id="1"]
[map slug="meu-mapa"]

Listar locais como cards

[list-locations id="1"]

7. Menus (Construtor de Menus)

Crie menus de navegação hierárquicos com sub-menus (dropdowns).

Como criar

1Admin → Menus → Criar Novo Menu

Preencha nome, slug e (opcional) hook de exibição

2Adicione links

  • Páginas dinâmicas
  • Posts do blog
  • Links personalizados

3Organize hierarquia

Use setas para: ↑↓ mover, → aninhar, ← recuar

4Salve Estrutura

Como exibir

Via hook (se configurado)

<x-hook name="public.main_menu" desc="Menu principal" />
Links inteligentes: se vincular a uma página, o menu atualiza automaticamente se o slug mudar!

8. Reactions (Curtidas)

Adiciona botões de reação (curtir/não curtir) ao final dos posts.

Como funciona

  • Aparece automaticamente via hook post.meta_end
  • Visitantes reagem sem login
  • Contadores atualizados via AJAX
  • 1 voto por IP (evita spam)

Configurações

Admin → Configurações → Leitura → Reactions:

  • Ativar/desativar reações
  • Tipo: 👍 Legal, ❤️ Coração, ⭐ Estrela
  • Permitir reação negativa
  • Reação única por visitante

Hook utilizado

post.meta_end

9. Share (Compartilhamento)

Botões de compartilhamento nas redes sociais ao final dos posts.

Redes suportadas

Rede Comportamento
WhatsApp Compartilhamento direto
Facebook Dialog de compartilhamento
X (Twitter) Composer com título+URL
LinkedIn Compartilhamento profissional
E-mail Abre cliente de e-mail

Como funciona

  • Aparece automaticamente via hook post.after_content
  • SVGs inline (sem bibliotecas externas)
  • Cores de marca no hover

Configurações

Admin → Configurações → Redes Sociais → Compartilhamento de Posts

Hook utilizado

post.after_content

Boas Práticas

Referenciando assets

CSS/JS de plugins

<link rel="stylesheet" href="{{ asset('plugins/comments/css/comments.css') }}">
<script src="{{ asset('plugins/comments/js/comments.js') }}"></script>

Verificando se plugin está ativo

Verificando view

@if(view()->exists('comments::comments-area'))
  {!! view('comments::comments-area', ['post' => $post])->render() !!}
@endif

Verificando classe

@if(class_exists('\Plugins\Avatars\Helpers\AvatarHelper'))
  <img src="{{ \Plugins\Avatars\Helpers\AvatarHelper::getUrl($user) }}">
@endif

Preferindo hooks

Recomendado: use hooks para integração automática. Se o plugin for desativado, o hook não renderiza nada — sem quebrar o tema.
