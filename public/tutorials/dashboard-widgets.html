Introdução

O Dashboard do Lunar Base não é uma página estática. Ele foi desenhado para ser extensível, permitindo que plugins e temas injetem seus próprios "boxes" (widgets) na tela inicial do administrador.

Este tutorial ensina como criar e registrar um box personalizado no Dashboard, desde a lógica (Controller) até a apresentação (View) e o registro no sistema.

Arquitetura do Sistema

Diferente de sistemas que pedem apenas "dados" para um widget, o Lunar Base utiliza uma abordagem baseada em Controllers e Views. Isso mantém a separação de responsabilidades (MVC) e permite que cada box tenha sua própria lógica de negócio.

1. Configuração

O arquivo /config/dashboard.php define o layout global (quantidade de colunas) e a view "moldura" padrão.

2. Classe Estática

A classe App\Support\Dashboard gerencia o registro, filtragem por permissão e renderização dos boxes.

3. Controllers

Cada box é servido por um método de controller que retorna uma View Blade já populada com dados.

Passo 1: Criar o Controller

O controller é responsável por buscar os dados e retornar a view do conteúdo do box. Ele não precisa estender nada especial, apenas o Controller padrão do Laravel.

Exemplo: Controller de Estatísticas

<?php

namespace App\Http\Controllers\Dashboard;

use App\Http\Controllers\Controller;
use App\Models\Post;
use App\Models\User;

class StatsController extends Controller
{
    public function index()
    {
        $totalPosts = Post::count();
        $totalUsers = User::count();

        // Retorna a view específica do conteúdo do box
        return view('admin.dashboard.boxes.stats', [
            'totalPosts' => $totalPosts,
            'totalUsers' => $totalUsers,
        ]);
    }
}
Dica: O método pode ter qualquer nome (ex: index, show, render), desde que você aponte corretamente no registro.

Passo 2: Criar a View

Crie a view que será injetada dentro da moldura do box. Esta view deve conter apenas o conteúdo interno (gráficos, listas, textos), sem o cabeçalho ou bordas do box.

Exemplo: resources/views/admin/dashboard/boxes/stats.blade.php

<div class="stats-grid">
    <div class="stat-card">
        <span class="stat-value">{{ $totalPosts }}</span>
        <span class="stat-label">Posts</span>
    </div>
    <div class="stat-card">
        <span class="stat-value">{{ $totalUsers }}</span>
        <span class="stat-label">Usuários</span>
    </div>
</div>

Passo 3: Registrar o Box

O registro é feito chamando o método estático Dashboard::add(). O melhor lugar para fazer isso é no método boot() do ServiceProvider do seu plugin ou tema.

Exemplo: No ServiceProvider

namespace Plugins\MeuPlugin;

use Illuminate\Support\ServiceProvider;
use App\Support\Dashboard;

class MeuPluginServiceProvider extends ServiceProvider
{
    public function boot(): void
    {
        // Registra o box no Dashboard
        Dashboard::add('meu-plugin-stats', [
            'title'      => 'Estatísticas Rápidas',
            'icon'       => 'bar-chart',
            'controller' => 'App\Http\Controllers\Dashboard\StatsController@index',
            'span'       => 2,
            'priority'   => 10,
            'permission' => 'view-dashboard',
        ]);
    }
}

Opções de Configuração

Ao chamar Dashboard::add('id', $config), você pode passar os seguintes parâmetros no array de configuração:

Parâmetro Tipo Padrão Descrição
title string — Título exibido no cabeçalho do box.
icon string null Nome de um ícone Lucide (ex: bar-chart, users). Opcional.
controller string — Classe e método no formato App\...\Controller@method.
span int 1 Quantas colunas o box ocupa no grid (1, 2, 3...).
priority int 100 Ordem de exibição. Menor número = maior prioridade (aparece primeiro).
permission string null Se definido, o box só aparece para usuários com essa permissão (RBAC).

Layout e CSS

O Dashboard utiliza CSS Grid para organizar os boxes. O número de colunas é definido em /config/dashboard.php (padrão: 3).

Como o span funciona

Se o seu grid tem 3 colunas e você define 'span' => 2, o box ocupará 2/3 da largura da linha. O CSS Grid usa auto-flow: dense, o que significa que boxes menores podem preencher automaticamente os espaços vazios deixados por boxes maiores.

Altura dos Boxes

Os boxes possuem align-self: start. Isso significa que eles não esticam verticalmente para igualar a altura do box vizinho. Se um box é mais alto que o outro, o espaço vazio fica abaixo do box menor, e não dentro dele.

Dica de CSS: Para que seu conteúdo interno ocupe bem o espaço, use height: 100% ou Flexbox dentro da sua view específica, mas evite definir alturas fixas rígidas.

Gerenciamento Avançado

A classe App\Support\Dashboard oferece outros métodos úteis para gerenciar boxes dinamicamente.

Remover um Box

Se você quiser ocultar um box registrado pelo core ou por outro plugin:

Dashboard::remove('system-welcome');

Modificar um Box Existente

Você pode alterar propriedades de um box já registrado (como o span ou a priority) sem precisar reescrevê-lo:

Dashboard::modify('system-welcome', function(array &$box) {
    $box['span'] = 3; // Faz o box de boas-vindas ocupar a linha toda
    $box['priority'] = 1; // Garante que ele seja o primeiro
});

Verificar se um Box Existe

if (Dashboard::has('meu-plugin-stats')) {
    // O box está registrado
}
