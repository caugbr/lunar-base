Como Funcionam os Hooks

Hooks são pontos de ancoragem (ganchos) posicionados nos arquivos de visão (Blade) do tema público ou do painel administrativo. Eles permitem que plugins injetem marcações HTML, estilos CSS, scripts Javascript ou alterem variáveis de forma transparente e desacoplada.

Declarando Ganchos (Temas)

Utilize o componente Blade <x-hook /> para disponibilizar um ponto de extensão. Os ganchos podem funcionar de duas maneiras:

1. Hook de Ação (Action - Sem conteúdo padrão)

Apenas reserva o espaço físico para que os plugins insiram marcação extra. É auto-fechado:

<x-hook name="post.footer_end" :params="['post' => $post]" />

2. Hook de Filtro (Filter - Com conteúdo de fallback)

Renderiza um bloco de conteúdo padrão caso nenhum plugin ou menu dinâmico esteja atrelado àquele ponto:

<x-hook name="public.main_menu" desc="Menu Principal">
    <!-- Fallback visual se nenhum plugin injetar conteúdo -->
    @foreach($menu as $item)
        <a href="{{ $item['href'] }}">{{ $item['label'] }}</a>
    @endforeach
</x-hook>

Registrando Listeners (Plugins)

No método boot() do Service Provider do seu plugin, registre um escutador para o gancho que deseja alimentar usando a classe HookManager:

PHPuse App\Support\HookManager;

// Registra a injeção ao final do conteúdo dos posts
HookManager::register('post.footer_end', function($params) {
    // $params recebe as variáveis injetadas na tag do hook
    $post = $params['post'] ?? null;

    if (!$post) return '';

    return view('meuplugin::area-comentarios', ['post' => $post])->render();
}, 'MeuPlugin'); // Rótulo identificador usado para depuração

Varredura & Autodescoberta

O Lunar Base possui um mecanismo inteligente que varre automaticamente os arquivos Blade do sistema em busca de tags <x-hook>, catalogando todos os ganchos físicos existentes.

Como a varredura funciona:

A classe HookDiscoverer realiza buscas recursivas nas pastas de themes/ (prioridade máxima), plugins/ e resources/views/ (core do sistema). Ela identifica se o gancho é um filtro ou ação, quais parâmetros ele disponibiliza e mapeia o arquivo físico onde ele reside.

Helper Útil para Painéis (Admin):

Você pode renderizar um dropdown unificado listando todos os ganchos descobertos para que o administrador escolha onde injetar um menu de navegação, por exemplo:

PHP{!! render_hooks_select([
    'name'        => 'hook',
    'id'          => 'menu_hook',
    'selected'    => $menu->hook ?? null,
    'placeholder' => '-- Selecione o local de exibição --'
]) !!}

Ganchos Nativos Disponíveis

Alguns dos ganchos já descobertos e utilizados pelos plugins oficiais:

Nome do Hook Setor Tipo Descrição
admin.header_user_avatar Admin Filtro Substitui o ícone do header administrativo pelo avatar circular do usuário.
profile.after_card Admin Ação Injeta formulários abaixo do card de perfil do usuário.
main.head Público Ação Dentro da tag <head> global.
main.before_content Público Ação Antes do conteúdo principal.
main.after_content Público Ação Depois do conteúdo principal.
public.main_menu Público Filtro Injeta o menu de navegação horizontal no cabeçalho.
post.meta_end Público (Post) Ação Ao final da área de metadados do artigo.
post.before_content Público (Post) Ação Antes do corpo de texto dos artigos do blog.
post.after_content Público (Post) Ação Após o corpo de texto dos artigos do blog.
post.footer_end Público (Post) Ação Ao final do rodapé do artigo.
