Introdução

Use o AdminMenu::add() no método boot() do seu ServiceProvider para injetar itens no menu administrativo do Lunar Base. O menu é renderizado automaticamente no layout admin, respeitando permissões e roles.

Sintaxe Básica

PHPuse App\Support\AdminMenu;

AdminMenu::add([
    'label'      => 'Meu Plugin',
    'icon'       => 'puzzle',           // Ícone Lucide
    'route'      => 'admin.meuplugin',  // Rota nomeada
    'active'     => 'admin.meuplugin.*',// Padrão de ativação
    'permission' => 'manage-pages',     // Permissão (opcional)
    'role'       => 'admin',            // Role (opcional)
], 'Taxonomias'); // Posição: após qual item?

Parâmetros

Parâmetro Obrigatório Descrição
label Sim Texto exibido no menu
icon Sim Ícone Lucide (ex: 'settings', 'users', 'file')
route Sim Nome da rota Laravel
active Sim Padrão para destacar item ativo (use * como wildcard)
permission Não Só aparece se usuário tiver esta permissão
role Não Só aparece se usuário tiver este role (admin, editor, etc)
Ícones Disponíveis: Visite Lucide Icons para ver todos os ícones disponíveis.

Exemplo Completo

No ServiceProvider do plugin

PHP// No método boot() do seu ServiceProvider
public function boot(): void
{
    $this->loadMigrationsFrom(__DIR__ . '/database/migrations');
    $this->loadViewsFrom(__DIR__ . '/resources/views', 'meuplugin');

    // Adiciona ao menu após "Taxonomias"
    \App\Support\AdminMenu::add([
        'label'      => 'Formulários',
        'icon'       => 'form',
        'route'      => 'admin.forms.index',
        'active'     => 'admin.forms.*',
        'permission' => 'manage-pages',
    ], 'Taxonomias');
}

Posicionamento

O segundo parâmetro do AdminMenu::add() define o item de referência para posicionamento. O novo item será inserido após o item especificado.

Itens do menu core (referência)

Dashboard
Páginas
Posts
Taxonomias
Mídia
Usuários
Configurações
Dica: Se o item de referência não existir, o novo item será adicionado no final do menu.

Boas Práticas

  • Use permissões: Sempre defina permission para não expor funcionalidades a usuários não autorizados.
  • Ícones consistentes: Use ícones Lucide que façam sentido semântico para a funcionalidade.
  • Rotas nomeadas: Sempre use rotas nomeadas no route para evitar quebras se URLs mudarem.
  • Wildcard no active: Use admin.meuplugin.* para que o item fique ativo em todas as sub-rotas.
