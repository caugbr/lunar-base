Introdução

Seu tema pode expor configurações cosméticas diretamente no painel administrativo geral do site. Quando o tema é ativado, esses campos de formulário são injetados no Admin; quando desativado, tornam-se inativos ou ocultos.

Registrando Configurações

No ThemeServiceProvider.php do seu tema, use a classe Settings para injetar campos:

Exemplo: Tema Astral Dark

PHPuse App\Support\Settings;

public function boot(): void
{
    // 1. Adiciona um divisor estético na aba Geral
    Settings::add([
        'type' => 'subtitle',
        'label' => 'Aparência: Astral Dark',
    ], 'general');

    // 2. Injeta um seletor de cores logo após o subtítulo
    Settings::add([
        'key' => 'astral_dark_accent_color',
        'type' => 'select',
        'options' => [
            'cyan' => 'Azul Celestial',
            'gold' => 'Dourado Estelar',
        ],
        'label' => 'Cor de Destaque',
        'description' => 'Escolha a cor principal para links e botões do site.',
        'default' => 'cyan',
    ], 'general', 'Aparência: Astral Dark');
}
Parâmetros do Settings::add():
  • $campoArray — Array com as propriedades do campo (key, type, label, etc.)
  • $group — Chave do grupo onde injetar (ex: 'general', 'reading')
  • $afterLabel — (Opcional) Rótulo âncora para posicionar o campo após um elemento existente

Usando no Layout do Tema

Recupere o valor salvo usando o helper setting() nas views Blade:

No layout do tema

BLADE<body data-accent="{{ setting('general.astral_dark_accent_color', 'cyan') }}">
    {{-- ... --}}
</body>

No CSS do tema (usando variável)

CSS:root {
    --accent-color: {{ setting('general.astral_dark_accent_color', 'cyan') == 'gold' ? '#ffd700' : '#00bcd4' }};
}
Sempre forneça fallback: O segundo parâmetro de setting() é o valor padrão caso a configuração esteja em branco ou o tema tenha sido desativado.

Comportamento ao Desativar

Quando o tema é desativado:

  • Os campos injetados desaparecem do painel de configurações
  • Os valores permanecem no banco de dados (não são apagados)
  • Se o tema for reativado, as configurações anteriores são restauradas

Boas Práticas

  • Prefixe as chaves: Use o nome do tema no key (ex: astral_dark_...) para evitar colisões com outras configurações.
  • Valores padrão seguros: Sempre defina default para que o tema funcione mesmo sem configuração manual.
  • Subtítulo identificador: Use um subtitle com o nome do tema para agrupar visualmente suas configurações no painel.
  • Não abuse: Injete apenas configurações cosméticas essenciais. Configurações complexas devem ter sua própria página no Admin.
