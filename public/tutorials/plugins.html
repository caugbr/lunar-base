Introdução

O sistema de plugins do Lunar Base CMS permite estender funcionalidades de forma modular e organizada. Cada plugin é autocontido e pode adicionar controllers, models, views, rotas, shortcodes, hooks e muito mais.

Menu Admin

Adicione itens ao menu administrativo de forma dinâmica e posicione onde desejar.

Shortcodes

Crie componentes reutilizáveis que podem ser inseridos em páginas e posts.

Hooks

Injete conteúdo em pontos específicos do site (head, before_content, after_content, etc).

Rotas Dinâmicas

Registre URLs customizadas sem modificar o core do sistema.

Models

Crie tabelas e relacionamentos usando Eloquent ORM.

Views & Assets

Templates Blade e arquivos estáticos com namespace próprio.

Estrutura de um Plugin

Um plugin típico no Lunar Base segue esta estrutura:

plugins/
└── NomeDoPlugin/
    ├── Http/
    │   └── Controllers/
    │       └── MeuController.php
    ├── Models/
    │   └── MeuModel.php
    ├── database/
    │   └── migrations/
    │       └── 2026_01_01_000000_create_tabela_table.php
    ├── resources/
    │   ├── views/
    │   │   ├── admin/
    │   │   └── public/
    │   └── assets/
    │       ├── css/
    │       └── js/
    ├── plugin.json
    ├── NomeDoPluginServiceProvider.php
    └── routes.php

Arquivos Principais:

Arquivo Descrição
plugin.json Manifesto do plugin (nome, versão, descrição, provider)
NomeDoPluginServiceProvider.php Classe de registro do plugin (carrega rotas, views, migrations)
routes.php Definição das rotas do plugin

Criando seu Primeiro Plugin

1Use o Comando Artisan

BASHphp artisan plugin:create "Nome do Plugin" "Descrição do plugin"

O comando fará perguntas interativas:

  • ✅ Requer banco de dados? (Cria Models e migrations)
  • ✅ Requer Controller e Rotas? (Cria estrutura HTTP)
  • ✅ Requer Views? (Cria diretório de views)

2Estrutura Gerada Automatically

O comando cria:

  • Diretórios necessários
  • Arquivo plugin.json com metadados
  • ServiceProvider configurado
  • Controller básico (se selecionado)
  • Model básico (se selecionado)
  • Migration em branco (se selecionado)
  • View de exemplo (se selecionado)
  • Link simbólico para assets em public/plugins/nome-do-plugin

3Ative o Plugin

Acesse Admin → Plugins e ative seu plugin. O sistema automaticamente:

  • Registra o ServiceProvider
  • Carrega migrations
  • Registra rotas
  • Disponibiliza views
Dica: Após criar o plugin, execute php artisan migrate para criar as tabelas no banco de dados.

Adicionando Itens ao Menu Admin

Use o AdminMenu::add() no método boot() do seu ServiceProvider para injetar itens no menu administrativo.

Sintaxe Básica:

PHPuse App\Support\AdminMenu;

AdminMenu::add([
    'label'      => 'Meu Plugin',
    'icon'       => 'puzzle',           // Ícone Lucide
    'route'      => 'admin.meuplugin',  // Rota nomeada
    'active'     => 'admin.meuplugin.*',// Padrão de ativação
    'permission' => 'manage-pages',     // Permissão (opcional)
    'role'       => 'admin',            // Role (opcional)
], 'Taxonomias'); // Posição: após qual item?

Parâmetros Explicados:

Parâmetro Obrigatório Descrição
label Sim Texto exibido no menu
icon Sim Ícone Lucide (ex: 'settings', 'users', 'file')
route Sim Nome da rota Laravel
active Sim Padrão para destacar item ativo (use * como wildcard)
permission Não Só aparece se usuário tiver esta permissão
role Não Só aparece se usuário tiver este role (admin, editor, etc)

Exemplo Completo:

PHP// No ServiceProvider do plugin
public function boot(): void
{
    $this->loadMigrationsFrom(__DIR__ . '/database/migrations');
    $this->loadViewsFrom(__DIR__ . '/resources/views', 'meuplugin');

    // Adiciona ao menu após "Taxonomias"
    \App\Support\AdminMenu::add([
        'label'      => 'Formulários',
        'icon'       => 'form',
        'route'      => 'admin.forms.index',
        'active'     => 'admin.forms.*',
        'permission' => 'manage-pages',
    ], 'Taxonomias');
}
Ícones Disponíveis: Visite Lucide Icons para ver todos os ícones disponíveis.

Configurações Dinâmicas (Settings)

O motor do Lunar Base permite que plugins estendam a página de configurações gerais de forma totalmente declarativa, registrando campos que geram formulários, validação e persistência de forma automatizada.

Métodos Disponíveis:

PHPuse App\Support\Settings;

// 1. Registra um novo grupo de configurações (nova aba ou assunto)
Settings::addGroup('meugrupo', [
    'title'       => 'Meu Plugin',
    'description' => 'Configurações de integração do meu plugin',
    'icon'        => 'cpu',
]);

// 2. Injeta um campo em um grupo existente (core ou adicionado acima)
Settings::add($campoArray, 'nome_do_grupo', 'RótuloReferência');

Parâmetros Explicados (Settings::add):

Parâmetro Obrigatório Descrição
$campoArray Sim Array contendo as propriedades do campo (key, type, label, description, etc.).
$group Sim Chave do grupo onde o campo será injetado (ex: 'reading', 'general').
$afterLabel Não String do rótulo âncora. O campo será posicionado logo após o campo que tiver esse título exato.
Âncora por Rótulo (afterLabel): O sistema de injeção de layout usa o texto do rótulo para determinar a posição. Isso permite colocar novos campos de forma fluida após elementos apresentacionais que não possuem chave no banco de dados, como os subtítulos (type => 'subtitle').

Exemplo Prático (Como usado no Plugin Reactions):

PHP// No método boot() do seu ServiceProvider

// 1. Injeta um subtítulo visual sem chave de banco de dados para criar um separador
\App\Support\Settings::add([
    'type' => 'subtitle',
    'label' => 'Plugin: Reactions',
], 'reading');

// 2. Injeta o switch funcional logo em seguida usando o rótulo do subtítulo como âncora
\App\Support\Settings::add([
    'key' => 'post_use_reaction',
    'type' => 'switch',
    'active' => 'Sim',
    'inactive' => 'Não',
    'label' => 'Usar reações nos posts',
    'description' => 'Habilita reações nos posts (Like)',
    'default' => true,
], 'reading', 'Plugin: Reactions');
Vantagem do Mecanismo: Qualquer configuração injetada através desse helper é integrada instantaneamente ao controlador de configurações, ganhando validação e gravação automática no banco de dados. Os valores podem ser resgatados em qualquer lugar do sistema via: setting('reading.post_use_reaction').

gistrando Shortcodes

Shortcodes permitem inserir conteúdo dinâmico em páginas e posts usando tags simples como [meu-shortcode].

Como Registrar:

PHPuse App\Helpers\ContentHelper;

// No método boot() do ServiceProvider
ContentHelper::registerShortcode('meu-componente', function($attributes, $content) {
    // $attributes = array de parâmetros passados no shortcode
    // $content = conteúdo entre [tag] e [/tag] (se houver)

    return view('meuplugin::componente', [
        'data' => $attributes,
        'content' => $content
    ])->render();
});

Usando Shortcodes:

Sintaxe Exemplo
Sem parâmetros [meu-componente]
Com parâmetros [meu-componente id="123" classe="destaque"]
Com conteúdo [meu-componente]Texto interno[/meu-componente]

Exemplo Prático - Shortcode de Formulário:

PHPContentHelper::registerShortcode('form', function($attributes, $content) {
    $slug = $attributes['slug'] ?? null;

    if (!$slug) {
        return '';
    }

    // Busca o formulário no banco
    $form = \Plugins\Forms\Models\Form::active()
        ->where('slug', $slug)
        ->first();

    if (!$form) {
        return '';
    }

    // Renderiza a view
    return view('forms::public.embed', ['form' => $form])->render();
});

Uso no editor:

[form slug="contato"]
Dica: O sistema automaticamente remove parágrafos envolvendo shortcodes (<p>[shortcode]</p> vira [shortcode]).

stema de Hooks

Hooks são pontos de injeção no tema onde plugins podem inserir conteúdo dinamicamente. É ideal para adicionar scripts, estilos, comentários, widgets, etc.

Hooks Disponíveis:

No Layout Principal (site-layout.blade.php):
BLADE<x-hook name="main.head" />           {{-- Dentro do <head> --}}
<x-hook name="main.before_content" /> {{-- Antes do conteúdo principal --}}
<x-hook name="main.after_content" />  {{-- Após o conteúdo principal --}}
Em Posts (post-templates/default.blade.php):
BLADE<x-hook name="post.before_content" :params="['post' => $post]" />
<x-hook name="post.after_content" :params="['post' => $post]" />
<x-hook name="post.footer_end" :params="['post' => $post]" />
<x-hook name="post.after_footer" :params="['post' => $post]" />
Em Páginas (templates/default.blade.php):
BLADE<x-hook name="page.before_content" :params="['page' => $page]" />
<x-hook name="page.after_content" :params="['page' => $page]" />
<x-hook name="page.after_footer" :params="['page' => $page]" />

Como Registrar um Hook:

PHPuse App\Support\HookManager;

// No método boot() do ServiceProvider
HookManager::register('post.footer_end', function($params) {
    // $params contém os dados passados pelo hook
    $post = $params['post'] ?? null;

    // Verifica se a view existe antes de renderizar
    if (view()->exists('meuplugin::componente')) {
        return view('meuplugin::componente', [
            'post' => $post
        ])->render();
    }

    return ''; // Retorna string vazia se não tiver conteúdo
}, 'NomeDoPlugin'); // Identificador para debug

Exemplo Prático - Sistema de Comentários:

PHP// No CommentsServiceProvider.php
\App\Support\HookManager::register('post.footer_end', function($params) {
    $post = $params['post'];

    // Verifica se a view existe
    if (view()->exists('comments::comments-area')) {
        return view('comments::comments-area', [
            'model' => $post
        ])->render();
    }

    return '';
}, 'Comments plugin');

Resultado: A área de comentários aparece automaticamente no final de todos os posts!

Dica: Em modo debug (APP_DEBUG=true), o sistema adiciona comentários HTML mostrando qual plugin injetou cada hook, facilitando o debugging.

tas Dinâmicas

O sistema de rotas dinâmicas permite registrar URLs customizadas que são resolvidas pelo RouteOrchestratorController quando nenhuma rota padrão é encontrada.

Como Registrar:

PHPuse App\Support\DynamicRoutes;

// No método boot() do ServiceProvider
DynamicRoutes::register('GET', 'portfolio/{slug}', function($slug) {
    // Retorna uma view ou chama um controller
    return app(\Plugins\Portfolio\Http\Controllers\PortfolioController::class)
        ->show($slug);
});

// Ou diretamente retornando uma view:
DynamicRoutes::register('GET', 'minha-pagina-customizada', function() {
    return view('meuplugin::pagina-customizada');
});

Parâmetros:

Parâmetro Descrição
method Método HTTP (GET, POST, PUT, DELETE, etc)
path Caminho da URL (use {param} para parâmetros dinâmicos)
resolver Callback que recebe os parâmetros e retorna a resposta

Exemplo 1 - Rota Simples:

PHPDynamicRoutes::register('GET', 'contato', function() {
    return view('meuplugin::contato');
});

Acessível em: https://seusite.com/contato

Exemplo 2 - Rota com Parâmetros:

PHPDynamicRoutes::register('GET', 'produto/{slug}', function($slug) {
    $produto = \Plugins\Loja\Models\Produto::where('slug', $slug)->firstOrFail();
    return view('loja::produto', ['produto' => $produto]);
});

Acessível em: https://seusite.com/produto/nome-do-produto

Exemplo 3 - Rota com Múltiplos Parâmetros:

PHPDynamicRoutes::register('GET', 'blog/{categoria}/{post}', function($categoria, $post) {
    return view('blog::post', [
        'categoria' => $categoria,
        'post' => $post
    ]);
});

Acessível em: https://seusite.com/blog/tecnologia/meu-post

Atenção: Rotas dinâmicas são resolvidas após as rotas padrão do sistema. Se houver conflito, a rota padrão tem prioridade.

Models e Relacionamentos

Plugins podem criar models Eloquent com relacionamentos polimórficos, permitindo vincular dados a Posts, Pages ou qualquer outro model do sistema.

Criando um Model:

PHPnamespace Plugins\MeuPlugin\Models;

use Illuminate\Database\Eloquent\Model;

class MeuModel extends Model
{
    protected $table = 'minha_tabela';

    protected $fillable = [
        'nome',
        'descricao',
        'user_id'
    ];

    protected $casts = [
        'created_at' => 'datetime',
        'updated_at' => 'datetime',
    ];

    // Relacionamentos
    public function user()
    {
        return $this->belongsTo(\App\Models\User::class);
    }
}

Relacionamento Polimórfico (Exemplo: Comentários):

PHP// No Model Comment
class Comment extends Model
{
    // Um comentário pertence a qualquer modelo (Post, Page, etc)
    public function commentable()
    {
        return $this->morphTo();
    }

    public function user()
    {
        return $this->belongsTo(\App\Models\User::class);
    }
}

// No Model Post (usando resolveRelationUsing no ServiceProvider)
Post::resolveRelationUsing('comments', function ($post) {
    return $post->morphMany(Comment::class, 'commentable')
        ->whereNull('parent_id')
        ->where('status', 'approved')
        ->orderBy('created_at', 'desc');
});

Usando Relacionamentos:

PHP// Criando comentário em um post
$post = Post::find(1);
$post->comments()->create([
    'user_id' => auth()->id(),
    'content' => 'Ótimo post!',
    'status' => 'approved'
]);

// Buscando comentários
$comments = $post->comments; // Retorna coleção de comentários

// Contando comentários
$count = $post->comments()->count();
Dica: Use resolveRelationUsing() para adicionar relacionamentos a models do core sem modificá-los diretamente. Isso mantém seu plugin compatível com atualizações.

Migrations e Banco de Dados

Migrations permitem versionar o esquema do banco de dados do seu plugin. Elas são automaticamente carregadas quando o plugin é ativado.

Criando Migration:

PHP// database/migrations/2026_01_01_000000_create_meus_items_table.php
use Illuminate\Database\Migrations\Migration;
use Illuminate\Database\Schema\Blueprint;
use Illuminate\Support\Facades\Schema;

return new class extends Migration {
    public function up(): void
    {
        Schema::create('meus_items', function (Blueprint $table) {
            $table->id();
            $table->string('nome');
            $table->text('descricao')->nullable();
            $table->foreignId('user_id')->constrained()->onDelete('cascade');
            $table->boolean('ativo')->default(true);
            $table->timestamps();
        });
    }

    public function down(): void
    {
        Schema::dropIfExists('meus_items');
    }
};

Relacionamentos Polimórficos:

PHPSchema::create('comentarios', function (Blueprint $table) {
    $table->id();

    // Cria commentable_id e commentable_type
    $table->morphs('commentable');

    $table->foreignId('parent_id')->nullable()
          ->constrained('comentarios')->onDelete('cascade');
    $table->foreignId('user_id')->nullable()
          ->constrained()->onDelete('cascade');
    $table->text('conteudo');
    $table->string('status')->default('pending');
    $table->timestamps();
});

Executando Migrations:

BASH# Migrar tudo (core + plugins)
php artisan migrate

# Migrar apenas plugins (se necessário)
php artisan migrate --path=plugins/NomeDoPlugin/database/migrations
Dica: O PluginServiceProvider automaticamente carrega migrations de todos os plugins quando você executa php artisan migrate.

Exemplo Prático: Plugin Comments

O plugin Comments demonstra múltiplas funcionalidades integradas:

1. ServiceProvider Completo:

PHPnamespace Plugins\Comments;

use Illuminate\Support\ServiceProvider;
use App\Models\Post;
use Plugins\Comments\Models\Comment;

class CommentsServiceProvider extends ServiceProvider
{
    public function boot(): void
    {
        // 1. Adiciona relacionamento polimórfico ao Post
        Post::resolveRelationUsing('comments', function ($post) {
            return $post->morphMany(Comment::class, 'commentable')
                ->whereNull('parent_id')
                ->where('status', 'approved')
                ->orderBy('created_at', 'desc');
        });

        // 2. Carrega resources
        $this->loadMigrationsFrom(__DIR__ . '/database/migrations');
        $this->loadRoutesFrom(__DIR__ . '/routes.php');
        $this->loadViewsFrom(__DIR__ . '/resources/views', 'comments');

        // 3. Registra hook para injetar área de comentários
        \App\Support\HookManager::register('post.footer_end', function($params) {
            $post = $params['post'];

            if (view()->exists('comments::comments-area')) {
                return view('comments::comments-area', [
                    'model' => $post
                ])->render();
            }

            return '';
        }, 'Comments plugin');
    }
}

2. Model com Relacionamentos:

PHPclass Comment extends Model
{
    protected $fillable = [
        'commentable_id',
        'commentable_type',
        'parent_id',
        'user_id',
        'author_name',
        'author_email',
        'content',
        'status',
    ];

    // Pertence a qualquer modelo (Post, Page, etc)
    public function commentable()
    {
        return $this->morphTo();
    }

    // Autor do comentário
    public function user()
    {
        return $this->belongsTo(\App\Models\User::class);
    }

    // Comentário pai (para respostas aninhadas)
    public function parent()
    {
        return $this->belongsTo(Comment::class, 'parent_id');
    }

    // Respostas aprovadas
    public function replies()
    {
        return $this->hasMany(Comment::class, 'parent_id')
            ->where('status', 'approved')
            ->orderBy('created_at', 'asc');
    }
}

3. Controller para Processamento:

PHPclass CommentController extends Controller
{
    public function store(Request $request)
    {
        $rules = [
            'commentable_id'   => 'required|integer',
            'commentable_type' => 'required|string',
            'parent_id'        => 'nullable|exists:comments,id',
            'content'          => 'required|string|min:3|max:1000',
        ];

        // Validação adicional para visitantes
        if (!Auth::check()) {
            $rules['author_name'] = 'required|string|max:255';
            $rules['author_email'] = 'required|email|max:255';
        }

        $validated = $request->validate($rules);

        // Moderação simples por palavras-chave
        $status = 'approved';
        $spamKeywords = ['buy', 'viagra', 'spam'];
        foreach ($spamKeywords as $keyword) {
            if (stripos($validated['content'], $keyword) !== false) {
                $status = 'pending';
                break;
            }
        }

        Comment::create([
            'commentable_id'   => $validated['commentable_id'],
            'commentable_type' => $validated['commentable_type'],
            'parent_id'        => $validated['parent_id'],
            'user_id'          => Auth::id(),
            'author_name'      => Auth::check() ? Auth::user()->name : $validated['author_name'],
            'author_email'     => Auth::check() ? Auth::user()->email : $validated['author_email'],
            'content'          => strip_tags($validated['content']),
            'status'           => $status
        ]);

        return back()->with('comment_status', 'Comentário enviado!');
    }
}

4. View com Recursão (Comentários Aninhados):

BLADE{{-- comments-area.blade.php --}}
<div class="comments-area">
    <h3>Comentários ({{ $model->comments()->count() }})</h3>

    <div class="comments-list">
        @forelse($model->comments as $comment)
            @include('comments::partials.comment-item', ['comment' => $comment])
        @empty
            <p>Sem comentários ainda.</p>
        @endforelse
    </div>

    <form action="{{ route('comments.store') }}" method="POST">
        @csrf
        <input type="hidden" name="commentable_id" value="{{ $model->id }}">
        <input type="hidden" name="commentable_type" value="{{ get_class($model) }}">
        <textarea name="content" required></textarea>
        <button type="submit">Comentar</button>
    </form>
</div>

{{-- partials/comment-item.blade.php --}}
<div class="comment-item">
    <div class="comment-author">{{ $comment->author_name }}</div>
    <div class="comment-text">{{ $comment->content }}</div>

    <button onclick="setReply({{ $comment->id }})">Responder</button>

    {{-- Respostas aninhadas recursivamente --}}
    @if($comment->replies->count() > 0)
        <div class="comment-replies">
            @foreach($comment->replies as $reply)
                @include('comments::partials.comment-item', ['comment' => $reply])
            @endforeach
        </div>
    @endif
</div>
O que aprendemos: Este exemplo mostra hooks, models polimórficos, relacionamentos recursivos, controllers, migrations, views recursivas e validação condicional!

O plugin Forms demonstra criação de CRUD admin, shortcodes dinâmicos e processamento de formulários:

PHPContentHelper::registerShortcode('form', function($attributes, $content) {

PHP\App\Support\AdminMenu::add([

3. Controller Admin com CRUD:

PHPclass FormsController extends Controller
{
    public function index(Request $request)
    {
        $query = Form::query();

        if ($request->filled('title')) {
            $query->where('title', 'like', '%' . $request->title . '%');
        }

        $forms = $query->withCount('submissions')
            ->orderBy('created_at', 'desc')
            ->paginate(setting('reading.pagination_max_items'));

        return view('forms::admin.forms.index', compact('forms'));
    }

    public function store(Request $request)
    {
        $validated = $request->validate([
            'title' => 'nullable|string|max:255',
            'slug' => 'required|string|unique:forms,slug|alpha_dash',
            'email_to' => 'nullable|email',
            'fields_schema' => 'required|json',
            'is_active' => 'boolean',
        ]);

        Form::create($validated);

        return redirect()->route('admin.forms.index')
            ->with('success', 'Formulário criado!');
    }
}

4. Model com Cast Automático:

PHPclass Form extends Model
{
    protected $fillable = [
        'slug', 'title', 'email_to', 'fields_schema',
        'is_active', 'submit_message'
    ];

    // Converte JSON automaticamente
    protected $casts = [
        'fields_schema' => 'array',
        'is_active'     => 'boolean',
    ];

    public function submissions()
    {
        return $this->hasMany(FormSubmission::class, 'form_id');
    }

    public function scopeActive($query)
    {
        return $query->where('is_active', true);
    }
}

5. Processamento de Submissão:

PHPclass GenericFormController extends Controller
{
    public function submit(Request $request, string $slug)
    {
        $form = Form::active()->where('slug', $slug)->firstOrFail();

        // Validação dinâmica baseada no schema
        $rules = [];
        foreach ($form->fields_schema as $field) {
            $rules[$field['key']] = $field['rules'] ?? 'nullable';
        }

        $validatedData = $request->validate($rules);

        // Salva submissão
        FormSubmission::create([
            'form_id'    => $form->id,
            'data'       => $validatedData,
            'ip_address' => $request->ip(),
        ]);

        // Envia email se configurado
        if (!empty($form->email_to)) {
            Mail::to($form->email_to)
                ->send(new FormSubmissionMail($form, $validatedData));
        }

        return back()->with('success', $form->submit_message);
    }
}

Boas Práticas

1. Nomenclatura:

  • Use PascalCase para nomes de plugins (MeuPlugin, Nao meu-plugin)
  • Namespaces sempre em Plugins\NomeDoPlugin
  • Slugs em kebab-case para URLs (meu-plugin)

2. Segurança:

  • Sempre valide inputs: $request->validate()
  • Use strip_tags() em conteúdos de usuários
  • Verifique permissões com userCan() ou middleware
  • Nunca exponha dados sensíveis em views

3. Performance:

  • Use @once para CSS/JS não duplicar
  • Eager loading: Post::with('comments')->get()
  • Verifique existência de views antes de renderizar
  • Use cache para dados frequentemente acessados

4. Organização:

  • Mantenha controllers focados (uma responsabilidade por controller)
  • Use partials para views reutilizáveis
  • Comente código complexo
  • Siga o padrão MVC rigidamente

5. Compatibilidade:

  • Use resolveRelationUsing() para não modificar models do core
  • Verifique se classes/funções existem antes de usar
  • Forneça fallbacks para quando recursos não estiverem disponíveis
Importante: Sempre teste seu plugin em um ambiente de desenvolvimento antes de publicar. Verifique compatibilidade com a versão do Laravel e do Lunar Base CMS.
